Password Generator

Application Overview

Website that generates a random password based on input and time of day. Inputs will use parameters such as using at least one
capital letter or a digit. It will also make use of a database full of phrases that can be mixed around in the generated password
as phrases are known to be much stronger when creating a password instead of random letters or numbers. Users will be able to create
an account on our website with a username and password, and the generation of passwords should be unique for every user so that an
adversary can't use this website to just generate passwords that someone coudl've gotten from this website and brute force their way
into any accounts they have set up with anybody else. This would be a big problem, and it's important that such a thing will never
happen in order to protect our users and make them feel confident in our product.

Technology Overview

This site will utilize a variation of the MEAN stack which includes MongoDB for document database management, Node.js,
Express, and HTML, CSS, and vanilla JavaScript (over Angular) on the client-side. Microsoft Azure is the cloud
computing service that will be employed for hosting and the project will be managed on GitHub with Git Version Control.

Feature List

    Simple user interface with various options
    Allow's restrictions and parameters like Length or minimum required amount of numbers or special characters
    different passwords generated from the same inputs at different times
    database used to pull from when creating the password
    server side algorithm that uses user input on a private variable to create a password

Future Feature List

Future features will potentially include saving passwords with an ecryption key generator.
Each user will be given an id and a key to reconstruct their passwords instead of saving them server side.
